But sometimes, I feel that if they could be commented, separated into blocks using whitespace, It could be simpler and less scary. So, I did a few searches on how to write better regexes and stumbled upon [this amazing answer](https://stackoverflow.com/a/60027277/15511745):

```js
//build regexes without worrying about
// - double-backslashing
// - adding whitespace for readability
// - adding in comments
let clean = (piece) => (piece
    .replace(/((^|\n)(?:[^\/\\]|\/[^*\/]|\\.)*?)\s*\/\*(?:[^*]|\*[^\/])*(\*\/|)/g, '$1')
    .replace(/((^|\n)(?:[^\/\\]|\/[^\/]|\\.)*?)\s*\/\/[^\n]*/g, '$1')
    .replace(/\n\s*/g, '')
);
window.regex = ({raw}, ...interpolations) => (
    new RegExp(interpolations.reduce(
        (regex, insert, index) => (regex + insert + clean(raw[index + 1])),
        clean(raw[0])
    ))
);
```

I liked this idea so much that I decided to package this into a library called [`betteregex`](https://www.npmjs.com/package/betteregex). Now, instead of this:

```js

const emailValidationRegex = /[a-z0-9!#$%&'*+/=?^_`{|}~-]+(?:\.[a-z0-9!#$%&'*+/=?^_`{|}~-]+)*@(?:[a-z0-9](?:[a-z0-9-]*[a-z0-9])?\.)+[a-z0-9](?:[a-z0-9-]*[a-z0-9])?/g
```

We can write this longer but more readable regex:

```js
const anythingInEmailRegex = '[a-z0-9!#$%&\'*+/=?^_`{|}~-]+';

const emailValidationRegex = regex`
	// RFC2822 Email validation

	// Match one or more alphabet, numbers, one of !#$%&'*+/=?^_{|}~-]+ or tildes
	${anythingInEmailRegex}

	// Open group
	(?:
		// Match a dot if any
		\.
		// Same as before
		${anythingInEmailRegex}
	// Close group, match one or more
	)*
	
	// The @
	@
	
	// Open group
	(?:
		// Provider name
		[a-z0-9](?:[a-z0-9-]*[a-z0-9])?
		// The dot
		\.
	// Close group
	)+

	/*
		The ending extension
		May not match everything because extensions are (mostly) letters
	*/
	[a-z0-9](?:[a-z0-9-]*[a-z0-9])?
${'g'}`
```

Those two examples are completely equal, but the second one is a bit more simple to understand. I'm sure that if the regex I showed you at the top would be readable if someone could write it like this.
